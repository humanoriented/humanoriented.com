Markup Done Right

The Problem

It's no secret: creating Web pages is easy! But if you ask most 'Web designers' what structural markup is, or why semantics are important they will probably snort, look at you funny and return to their Dreamweaver layout. And if you ask them to build you a Web site, chances are they'll create one that looks nice but is riddled with problems due to a lack of understanding of how browsers and http work, and the importance of standards-compliant markup.

Here's what the site looked like in Photoshop, created by a very competent graphic designer:

This is actually a slightly complicated layout. In addition, there are three tabs that required a 'hover' effect and each section of the site needed to have certain tabs highlighted. The buttons also required a 'hover' effect.

The client came to me asking for some CSS 'engineering,' as their Web designers just couldn't figure out how to make certain layout changes to their site without breaking everything else. The existing site used a lot of javascript and images for the hover effects; complex, ugly nested markup; non-externalized style declarations; completely lacking in alt and title attributes and other semantically meaningful markup; and used tiled images that resulted in numerous http requests just to load this simple screen.

If you don't recognize the numerous problems with this, you're probably a normal person. But to a real developer, this thing is a nightmare waiting to be woken up from. Rather than make the simple changes required I decided to fix the site so ongoing maintenance wouldn't cause my brain to leak out of my ears.

The Solution

Rather than rag on about the problems, here's a list of things I changed:

  • Externalize all styles
  • Make all style, id and class names meaningful
  • Reduce the number of http requests, and speed up the site
  • Replace all the strange markup with a simple structure
  • Wrap all content in semantically meaningful markup (eg, headers get <h1>, lists get <ul>, etc)
  • Replace javascript hover effects with pure css, sprite-based approaches
  • 'SEO' the content and markup appropriately
